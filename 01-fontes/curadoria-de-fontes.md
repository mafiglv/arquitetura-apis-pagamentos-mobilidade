# 📚 Curadoria de Fontes

## 🎯 Objetivo da curadoria

Esta seção reúne as fontes utilizadas no NotebookLM para construir o caderno temático sobre **Arquitetura de APIs para Pagamentos Digitais e Mobilidade**.

A curadoria foi organizada com o objetivo de estudar como APIs, microsserviços, arquitetura orientada a eventos, segurança e sistemas de pagamento se conectam em soluções digitais de alta criticidade.

O foco do estudo foi compreender como uma arquitetura de back-end pode apoiar fluxos como:

* registro de eventos de mobilidade;
* validação de clientes;
* processamento de transações;
* prevenção de cobranças duplicadas;
* integração entre serviços;
* segurança de dados;
* rastreabilidade e observabilidade.

## 🧠 Critérios de seleção das fontes

As fontes foram selecionadas considerando os seguintes critérios:

* relação direta com arquitetura de APIs;
* aplicação em sistemas corporativos e distribuídos;
* abordagem sobre microsserviços;
* conteúdo sobre arquitetura orientada a eventos;
* relação com pagamentos digitais e transações financeiras;
* boas práticas de segurança em APIs;
* confiabilidade técnica da fonte;
* potencial de aplicação em cenários de mobilidade e pagamentos.

## 🔗 Fontes selecionadas

### 1. [API-First Architecture: Quando Construir um Ecossistema Digital Faz Sentido](https://www.guilhermefavaron.com.br/p/api-first-architecture-quando-faz-sentido)

**Tema principal:**
Arquitetura API-First e ecossistemas digitais.

**Justificativa:**
O material apresenta a API como elemento central na construção de produtos digitais, mostrando que ela vai além de um recurso técnico. A leitura contribuiu para entender como APIs podem conectar sistemas, parceiros e canais digitais, além de reforçar a importância de planejar a comunicação entre aplicações antes mesmo da interface final.

### 2. [Práticas recomendadas para design de API Web RESTful — Microsoft Learn](https://learn.microsoft.com/pt-br/azure/architecture/best-practices/api-design)

**Tema principal:**
Design de APIs REST e boas práticas de arquitetura.

**Justificativa:**
A documentação traz recomendações técnicas para desenvolver APIs Web mais claras, padronizadas e fáceis de consumir. Foi útil para compreender práticas como organização de recursos, uso adequado de métodos HTTP, versionamento, tratamento de respostas e documentação.

### 3. [Arquitetura de microsserviços — .NET | Microsoft Learn](https://learn.microsoft.com/pt-br/dotnet/architecture/microservices/architect-microservice-container-applications/microservices-architecture)

**Tema principal:**
Microsserviços e arquitetura distribuída.

**Justificativa:**
O conteúdo explica como sistemas maiores podem ser divididos em serviços menores, independentes e especializados. Esse conceito ajudou a relacionar a arquitetura de microsserviços com um fluxo de pagamentos e mobilidade, no qual diferentes serviços podem cuidar de etapas como validação de cliente, cálculo de valor, registro de transação e cobrança.

### 4. [O que é EDA? — Event-Driven Architecture Explained | AWS](https://aws.amazon.com/pt/what-is/eda/)

**Tema principal:**
Arquitetura orientada a eventos.

**Justificativa:**
A explicação da AWS ajudou a compreender como sistemas podem reagir a eventos, como “passagem registrada”, “pagamento aprovado” ou “cobrança gerada”. Esse conteúdo foi importante para entender o papel da comunicação assíncrona em arquiteturas mais flexíveis, escaláveis e preparadas para picos de demanda.

### 5. [Guidance for Building Payment Systems Using Event-Driven Architecture on AWS](https://aws.amazon.com/solutions/guidance/building-payment-systems-using-event-driven-architecture-on-aws/)

**Tema principal:**
Sistemas de pagamento com arquitetura orientada a eventos.

**Justificativa:**
Esse guia conecta diretamente pagamentos digitais com arquitetura orientada a eventos. Ele ajudou a visualizar como um sistema de pagamento pode processar transações, registrar operações, lidar com falhas e manter resiliência em fluxos financeiros. Também contribuiu para aproximar o estudo de um cenário de alta criticidade, em que segurança, consistência e rastreabilidade são essenciais.

### 6. [OWASP API Security Project — OWASP Foundation](https://owasp.org/www-project-api-security/)

**Tema principal:**
Segurança em APIs.

**Justificativa:**
A OWASP é uma referência importante para entender riscos e vulnerabilidades em APIs. O material apoiou a identificação de problemas como falhas de autenticação, falhas de autorização, exposição de dados sensíveis e consumo excessivo de recursos. No contexto de pagamentos digitais e mobilidade, essa fonte reforçou a importância de proteger dados, controlar acessos e reduzir riscos técnicos.


## 🧩 Síntese da curadoria

As fontes selecionadas permitiram construir uma visão integrada sobre arquitetura de APIs em sistemas de pagamentos digitais e mobilidade.

A partir delas, foi possível estudar:

* API-First;
* APIs REST;
* microsserviços;
* arquitetura orientada a eventos;
* segurança em APIs;

* idempotência;
* observabilidade;
* escalabilidade;
* resiliência;
* rastreabilidade de transações.
